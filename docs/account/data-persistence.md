# Data Persistence Design: Create User Account

## Database Technology
PostgreSQL

## Persistence Objective
Store user account data linked to an existing user profile, enforce username uniqueness, and persist the linked profile email for login compatibility.

## Main Tables
- `user_account`
- `user_profile`

## Table Definition
```sql
create table user_account (
  user_account_id bigserial primary key,
  email varchar(255) unique not null,
  password_hash varchar(255) not null,
  account_status varchar(30) not null default 'ACTIVE',
  created_at timestamp not null default current_timestamp,
  username varchar(255) unique,
  profile_id bigint references user_profile(profile_id),
  role varchar(30) not null default 'Fundraiser'
);

create table user_profile (
  profile_id bigserial primary key,
  name varchar(255) not null,
  email varchar(255) unique not null,
  phone_num varchar(50) not null,
  address varchar(500) not null,
  created_at timestamp not null default current_timestamp
);
```

## Column Descriptions
- `user_account_id`: Unique identifier for the account, auto-generated by PostgreSQL sequence.
- `email`: Email copied from the linked `user_profile` record. Required for the current login flow.
- `password_hash`: Bcrypt hashed password.
- `account_status`: Current account state such as `ACTIVE`.
- `created_at`: Timestamp showing when the account was created.
- `username`: Unique username used for account creation rules.
- `profile_id`: Foreign key linking the account to `user_profile.profile_id`.
- `role`: User role such as `Fundraiser`, `Donee`, `User admin`, or `Platform manager`.

## Queries Required for Create Account

### Check if username exists
```sql
select 1 from user_account where username = ?;
```

### Find profile by ID
```sql
select profile_id, name, email, phone_num, address
from user_profile
where profile_id = ?;
```

### Insert new account
```sql
insert into user_account (profile_id, email, username, password_hash, role)
values (?, ?, ?, ?, ?);
```

## Persistence Rules
- `username` must be unique across all user accounts.
- `profile_id` must reference an existing row in `user_profile`.
- `email` cannot be null in `user_account`, so the create account flow must copy the email from the linked profile.
- Passwords are stored only as bcrypt hashes, never as plain text.
- The create account use case performs a username check, a profile lookup, and then an insert.

## Mapping to BCE
- Boundary does not access the database directly.
- Controller coordinates validation, uniqueness checking, profile lookup, and account creation.
- Entities perform PostgreSQL lookup and insert operations.
- PostgreSQL is the system of record for account and profile data.
