# Data Persistence Design: Create Fundraising Activity

## Database Technology
PostgreSQL

## Persistence Objective
Store new fundraising activity records with the title, description, target amount, category, start date, end date, and lifecycle status needed for later viewing and management.

## Main Table
- `fundraising_activity`

## Table Definition
```sql
create table fundraising_activity (
  activity_id bigserial primary key,
  title varchar(255) not null,
  description text not null,
  target_amount decimal(12, 2) not null,
  category varchar(100) not null,
  start_date date not null,
  end_date date not null,
  status varchar(30) not null default 'ACTIVE',
  created_at timestamp not null default current_timestamp
);
```

## Column Descriptions
- `activity_id`: Unique identifier for the fundraising activity, generated by PostgreSQL.
- `title`: Activity title shown to users.
- `description`: Detailed explanation of the fundraising purpose.
- `target_amount`: Requested fundraising amount stored as `DECIMAL(12,2)`.
- `category`: Activity category label.
- `start_date`: Date the fundraising activity begins.
- `end_date`: Date the fundraising activity ends.
- `status`: Activity state, defaulting to `ACTIVE`.
- `created_at`: Timestamp of record creation.

## Queries Required for Create Fundraising Activity

### Insert new fundraising activity
```sql
insert into fundraising_activity (title, description, target_amount, category, start_date, end_date)
values (?, ?, ?, ?, ?, ?);
```

## Persistence Rules
- The boundary validates required fields before the backend call.
- `target_amount` must be greater than zero.
- `end_date` must be after `start_date`.
- `status` defaults to `ACTIVE` when a new record is inserted.
- The entity persists the activity by calling `FundraisingActivity.saveFundraisingActivity(...)`.

## Mapping to BCE
- Boundary does not access the database directly.
- Controller coordinates the create activity request and save decision.
- Entity performs the PostgreSQL insert.
- PostgreSQL is the system of record for fundraising activity data.
